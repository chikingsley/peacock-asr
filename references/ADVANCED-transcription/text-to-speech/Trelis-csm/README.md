# Trelis CSM-1B

>![TIP]
>This is based on the original repo available at https://github.com/SesameAILabs/csm

Text to Speech with Sesame's CSM model, including:
- Context-aware Text to Speech
- Voice cloning

Note that fine-tuning is not supported. See `archive` for some incomplete scripts.

## Getting started

Run
```bash
cd text-to-speech/Trelis-csm
uv venv --python 3.10.0
uv pip install -r requirements.txt
```

## Usage

You can run this model on CPU or on a remote service such as Runpod (one-click template [here, affiliate link](https://www.runpod.io/console/deploy?template=ifyqsvjlzj&ref=jmfkcdio)).

### Basic Text-to-Speech

For basic text-to-speech synthesis with the default voice:

```bash
uv run python csm-tts.py
```

This will generate a simple audio file called `audio.wav` with the default message "Hello from Sesame." using the model's default voice.

### Voice Cloning

To clone a voice using sample recordings:

```bash
uv run clone.py
```

This script uses example audio files in the `data` directory to clone a voice and generate new speech with the same voice characteristics.

To clone your own voice:
1. Record yourself speaking and place the m4a or wav file into the data folder. If you recorded an m4a file, run `uv run convert_m4a_to_wav.py` to convert it to wav.
2. Write out the transcription of your voice within `clone.py`. Use a transcription service if needed.
3. Adjust the texNo. 32, Woodview, Mount Merrion Avenue 11.m4at you wish to generate in `clone.py`.
4. Run `uv run clone.py`

The script will generate two outputs files: a) a baseline voice without cloning, and b) a cloned version of your voice.

### Context-Aware Text-to-Speech 

For a more advanced usage with context-aware generation and interactive recording:

```bash
uv run context-aware-tts.py
```

This script demonstrates a multi-turn conversation where:
1. The model generates initial speech with its default voice
2. You then can record your voice reading out specific text.
3. The model generates responses (both AI and a clone of your voice) that maintain contextual awareness of previous turns

Additional options:
```bash
uv run context-aware-tts.py --output_dir path/to/output --human_recording path/to/recording.wav
```

### Playing Complete Conversations

To play back a complete conversation generated by context-aware-tts.py:

```bash
uv run play_conversation.py
```

This will play all the audio files from a conversation in sequence, simulating a complete natural conversation flow. The script handles both real human recordings and generated dummy responses.

Options include:
```bash
# Specify a different conversation directory
uv run play_conversation.py --dir path/to/conversation

# Display transcripts alongside audio playback
uv run play_conversation.py --with_transcript

# Adjust the pause between turns
uv run play_conversation.py --pause 1.5
```

### Converting M4A to WAV

If you have recordings in M4A format, it's recommended to convert them to WAV format for best compatibility:

```bash
uv run convert_m4a_to_wav.py
```

This will automatically convert all M4A files in the `data` directory to WAV format while preserving the original files.
